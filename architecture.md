# Bluebook Simulator PWA Architecture

## 1. Overview & Goal
The Bluebook Simulator is a Progressive Web App (PWA) designed to perfectly replicate the College Board's Bluebook testing environment locally on school-issued hp-elitebook. likely won't work on other devices

The app primarily utilizes an **AI-driven JSON workflow** combined with **plain-text rendering libraries** (for math, graphs, and simple diagrams). However, to fully support complex anatomical or historical diagrams without needing a backend server, the app also features a **local image injection** system where users can drag-and-drop screenshots directly into the app alongside their JSON data.

---

## 2. Technology Stack
* **Core Framework:** React 18 + TypeScript (built with Vite)
* **PWA Engine:** `vite-plugin-pwa` (handles service workers, offline caching, and native installation prompts)
* **Styling:** Vanilla CSS / CSS Modules (to meticulously replicate the Bluebook design system: exact typography, colors, borders, and layouts)
* **Content Rendering Libraries:**
  * **Math:** `KaTeX` (via `react-katex`) for ultra-fast LaTeX math equation rendering.
  * **Graphs/Calculus:** `function-plot` for plotting mathematical functions dynamically.
  * **Diagrams/Biology/Logic:** `Mermaid.js` for rendering complex flowcharts, biological cycles, or logic trees from text.
  * **Geometry/Shapes:** Raw `SVG` (sanitized via `DOMPurify`) for custom geometric figures.
  * **Rich Text:** Markdown parser (e.g., `marked`) to handle bold, italic, and structure.

---

## 3. User Workflow
1. **Exam Type Selection:** User opens the PWA and selects the exam they want to practice (e.g., AP Biology).
2. **Question Input Stage (Split-Screen):** The user is taken to a data ingestion view.
   * **Left Half (JSON Input):** The user pastes the JSON generated by the AI (using a provided prompt) into a large text box.
   * **Right Half (Image Dropzone):** Once the JSON is pasted, the app instantly parses it. If the AI determined it couldn't render certain graphs or diagrams, it flags them in the JSON. The app displays a checklist on the right half of the screen (e.g., *"Question 4 requires an image. Please screenshot it and paste/drop it here."*).
3. **Image Attachment:** The user takes screenshots of the missing diagrams from their physical test/PDF and pastes or drops them into the right side of the screen. The app locally links these images (`Blob` URLs) to the respective questions.
4. **Pre-flight & Customization:** Once all required images are provided, the app displays test metadata (e.g., "Questions: 60 | Suggested Time: 90 mins"). The user can customize the timer and click **"Start Exam"**.
6. **Simulation Mode:** The app enters full-screen mode, rendering a 1:1 replica of the Bluebook interface.

---

## 4. UI/UX Design (The Bluebook Clone)
The simulation view must feel identical to the real Bluebook app. Key components include:
* **Header Bar:** 
  * Left: Section directions.
  * Center: Hideable Countdown Timer (turns red at 5 minutes).
  * Right: Tools (Annotate, More).
* **Main Stage (Split Pane for Reading, Single Pane for Math):**
  * **Left Pane (Stimulus):** The passage, graph, or diagram.
  * **Right Pane (Question):** The specific question and answer choices.
* **Interactive Tools:**
  * Answer Eliminator (strike-through choices).
  * "Mark for Review" flag.
  * Text highlighter / annotation tool.
* **Footer Navigation:**
  * Grid ribbon of question numbers at the bottom.
  * Dashed outlines for unanswered, solid for answered, flagged icon for review.
  * "Back" and "Next" buttons.

## 5. The "Bluebook" JSON Schema
To render text into rich diagrams, the AI must output a specific format. Here is a rough draft of the JSON structure the app expects:

```json
{
  "metadata": {
    "title": "AP Calculus AB Practice",
    "examType": "AP",
    "totalQuestions": 45,
    "suggestedTimeMinutes": 105
  },
  "questions": [
    {
      "id": "q1",
      "stimulus": {
        "type": "function-plot",
        "data": "f(x) = x^2 - 4x + 3"
      },
      "text": "What are the roots of the function shown above, where $$f(x) = 0$$?",
      "options": [
        { "id": "A", "text": "$$x = 1$$, $$x = 3$$" },
        { "id": "B", "text": "$$x = -1$$, $$x = -3$$" },
        { "id": "C", "text": "$$x = 2$$, $$x = 4$$" },
        { "id": "D", "text": "$$x = 0$$, $$x = 3$$" }
      ],
      "correctAnswer": "A",
      "explanation": "Factoring the equation gives $$(x-1)(x-3) = 0$$."
    },
    {
      "id": "q2",
      "stimulus": {
        "type": "mermaid",
        "data": "graph TD;\n A[DNA] -->|Transcription| B(RNA);\n B -->|Translation| C{Protein};"
      },
      "text": "Based on the biological diagram above, what process is represented by the arrow moving from RNA to Protein?",
      "options": [
        { "id": "A", "text": "Replication" },
        { "id": "B", "text": "Transcription" },
        { "id": "C", "text": "Translation" },
        { "id": "D", "text": "Mutation" }
      ],
      "correctAnswer": "C"
    },
    {
      "id": "q3",
      "stimulus": {
        "type": "image",
        "data": "diagram_1.png"
      },
      "text": "Identify the structure labeled X in the provided diagram.",
      "options": [
        { "id": "A", "text": "Mitochondria" },
        { "id": "B", "text": "Chloroplast" },
        { "id": "C", "text": "Nucleus" },
        { "id": "D", "text": "Ribosome" }
      ],
      "correctAnswer": "B"
    }
  ]
}
```

## 6. Rendering Engine Logic
The core of the app is a Dynamic Content Renderer component that interprets the strings provided in the JSON:
* **Double Dollar Signs (`$$...$$`):** Intercepted and passed to KaTeX for math rendering.
* **Stimulus type `mermaid`:** Injected into a Mermaid.js initialization container.
* **Stimulus type `function-plot`:** Parsed and rendered onto an HTML5 canvas.
* **Stimulus type `svg`:** Safely injected using `DOMPurify` to ensure no XSS vulnerabilities while allowing complex geometric shapes.
* **Stimulus type `image`:** Maps the filename provided in the JSON to the `Blob` URL generated from the user's drag-and-drop upload, rendering it seamlessly in an `<img>` tag.

---

## 7. PWA Offline Strategy
* **Service Worker:** Caches the entire React bundle, CSS, KaTeX fonts, and external libraries (like Mermaid) upon first load.
* **Storage:** Any "in-progress" exams or saved JSON files are stored in the browser's `IndexedDB` or `localStorage`, ensuring that if a student accidentally closes the EliteBook, their timer and answers are perfectly preserved when they reopen the PWA.
